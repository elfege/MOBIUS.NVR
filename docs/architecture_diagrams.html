6. Credential Encryption Flow

sequenceDiagram
    participant UI as Browser UI
    participant Route as routes/camera.py
    participant Cred as CredentialDBService
    participant DB as camera_credentials

    Note over UI,DB: Save Credentials
    UI->>Route: PUT /api/camera/{serial}/credentials
{username, password, scope}
    Route->>Cred: store_credential(key, user, pass, vendor, type)
    Cred->>Cred: _get_encryption_key()
SHA-256(NVR_SECRET_KEY)
    Cred->>Cred: _encrypt(username) → AES-256-GCM → base64
    Cred->>Cred: _encrypt(password) → AES-256-GCM → base64
    Cred->>DB: POST /camera_credentials
{credential_key, username_enc, password_enc}
    alt 409 Conflict
        Cred->>DB: PATCH /camera_credentials
WHERE key=serial AND type=scope
    end
    DB-->>Cred: 200 OK
    Cred-->>Route: True
    Route-->>UI: {success: true}

    Note over UI,DB: Generate go2rtc.yaml (startup)
    participant Gen as generate_go2rtc_config.py
    participant SHM as /dev/shm (tmpfs)
    Gen->>DB: SELECT credential_key, username_enc, password_enc
    Gen->>Gen: _decrypt(username_enc) → plaintext
    Gen->>Gen: _decrypt(password_enc) → plaintext
    Gen->>Gen: resolve ${go2rtc_username} → plaintext
    Gen->>SHM: Write go2rtc.yaml (RAM only, never disk)
