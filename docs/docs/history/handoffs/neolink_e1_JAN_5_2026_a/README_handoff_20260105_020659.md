### 2. Fixed ONVIF PTZ Handler Crash

**Problem:** E1 camera has `onvif_port: null` (correct - E1 doesn't support ONVIF). The ONVIF PTZ handler was passing `None` to `ONVIFClient.get_camera()` which crashed on `int(None)`.

**Error:**
```
ERROR:services.onvif.onvif_client:Unexpected error connecting to 192.168.10.123:None - int() argument must be a string, a bytes-like object or a real number, not 'NoneType'
```

**Fix:** Added null check in 5 methods in `services/onvif/onvif_ptz_handler.py`:
- `ptz_move` (line ~100)
- `get_presets` (line ~197)
- `goto_preset` (line ~280)
- `set_preset` (line ~365)
- `remove_preset` (line ~440)

Now gracefully returns `"Camera does not support ONVIF"` instead of crashing.

**Commit:** `104ed26` - Fix ONVIF PTZ handler crash when onvif_port is null
